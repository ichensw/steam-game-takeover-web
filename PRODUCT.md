# Product

## Register

product

## Users

The primary users are administrators who manage the Steam game takeover community. They need to review takeovers, users, feedback, reports, whitelist access, system settings, and operational status with minimal friction.

## Product Purpose

This web admin system gives operators a reliable control panel for the "兔兔窝接龙" service. Success means administrators can find records quickly, act on them confidently, and understand current platform activity without switching tools or relying on raw API responses.

## Brand Personality

Game-community, controlled, efficient.

The interface should feel like a practical community operations console: structured and trustworthy, with enough gaming-community energy to avoid looking like a generic enterprise template.

## Anti-references

Avoid the mini program's pink-purple rabbit visual style as the main admin identity. Avoid marketing-page composition, decorative card grids, oversized hero sections, glassmorphism, heavy gradients, and playful visuals that reduce operational clarity.

## Design Principles

1. Prioritize operator speed: filters, tables, detail views, and status actions should be easy to scan and repeat.
2. Keep community context visible: show enough user, Steam, KOOK, and takeover context so admins can make decisions without opening multiple screens.
3. Use game-console character sparingly: let navigation, status color, and small accents carry the identity instead of decoration.
4. Make state explicit: pending, active, ended, blocked, whitelisted, adopted, and ignored states should be visually distinct and textually clear.
5. Prefer boring reliability: standard controls, predictable layouts, and consistent admin flows beat novelty.

## Accessibility & Inclusion

Target WCAG AA contrast. Support keyboard navigation for forms, filters, menus, dialogs, and table actions. Keep motion short and state-driven, and respect reduced-motion preferences. Do not rely on color alone to communicate status.
